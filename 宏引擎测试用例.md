## 实验性宏引擎说明

* 实验性宏引擎：兼容部分sillytavern的实验性宏引擎语法，详情查看[ST官方文档](https://docs.sillytavern.app/usage/core-concepts/macros/#double-colon-separator)
  * 实验性宏引擎默认关闭，可在右上角设置中的工具设置界面顶部的开关处开启宏引擎功能。
  * 变量管理：上滑发送按钮，点击出现的`{ }`按钮进入变量管理界面。
  * 支持包括变量、时间日期、随机、运算、逻辑、if else条件等，不常用的类似`{{personality}}`的占位符不支持。

### 空格和换行
用于在字符串变量的前后添加空格和换行（setvar时使用）。
* {{newline}}	插入换行符
* {{newline::count}}	插入多个换行符
* {{space}}	插入空格字符
* {{space::count}}	插入多个空格

保留空白：
当您需要保留作用域内容中的所有空白字符（包括前导/尾随换行符和缩进）时，请使用`#`标志及以下写法：
```
{{ # setvar code }}
    function hello() {
        return "world";
    }
{{ /setvar }}
```
### 变量名和简写
局部变量操作的简写形式。例如读取局部变量：{{.myvar}}
全局变量操作的简写形式。例如读取全局变量：{{$myvar}}
变量名遵循与宏标识符相同的规则：以字母开头，后跟字母、数字、下划线或连字符。最后一个字符不能是下划线或连字符。
如果您的变量标识符不符合标准规则，则必须使用完整的变量宏语法（例如{{getvar::my§var----}}）

### 时间日期
* {{time}}	当前当地时间
* {{time::UTC±(offset)}}	带UTC偏移量的时间
* {{date}}	当前本地日期（简写）
* {{weekday}}	当前星期几

### 随机
{{roll::1d20}}等。

### 转义
`\{\{getvar::var\}\}`
用\进行转义，会保留原文而不执行宏。

### forksilly特有宏
`{{initvar::var::value}}`：使用initvar进行变量的初始化，只会在变量不存在时执行；避免在使用setvar时每次执行都会重新将变量设为初始值。请按需使用setvar和initvar。

## 说明

1. 所有宏都必须以{{开头且后面是宏标识符，否则不会被识别为宏。例如，{{100 || .myvar}}这样的写法是无效的，因为100不是任何宏标识符。
2. 发送消息时，除聊天列表中的内容外，所有提示词（角色卡内容、user描述、世界书、预设等）都会被宏引擎处理并立即更新变量。
3. 用户当前发送的消息（从输入框发送的内容）不会触发实验宏的执行；修改历史消息的内容并保存，宏会模拟执行，但不会实际更新变量值。如果需要手动更改变量，请使用变量管理面板（输入按钮上划）。
4. 提示词预览功能不会触发变量的实际更新，仅模拟执行结果。
5. 接收新消息：收到AI回复的消息时，会立即对新消息进行处理，更新变量。
6. 重新生成功能会回滚上一次的变量快照重新计算；但如果离开聊天再回来点击重新生成，则不会回滚，可能造成数值累加，需注意。

**Variable Shorthand Operators**

*与完整的操作方法有一些区别，快捷语法在进行变量修改后通常不会返回变量值。*

变量操作的快捷语法：

---

### 获取变量 (Get Variable)
使用简单的前缀即可获取变量值：

* `{{.myvar}}` // 获取**局部**变量 "myvar"
* `{{$myvar}}` // 获取**全局**变量 "myvar"

等同于 `{{getvar::myvar}}` 和 `{{getglobalvar::myvar}}`。

---

### 设置变量 (Set Variable)
使用 `=` 运算符设置变量值：

* `{{ .myvar = Hello World }}` // 设置局部变量
* `{{ $myvar = Some value }}`  // 设置全局变量

等同于 `{{setvar::myvar::Hello World}}` 和 `{{setglobalvar::myvar::Hello World}}`。执行后返回空字符串。

---

### 递增 (Increment)
使用 `++` 将数字变量增加 1：

* `{{.counter++}}` // 递增局部变量，返回空字符串
* `{{$counter++}}` // 递增全局变量，返回空字符串

类似 `{{incvar counter}}` 和 `{{incglobalvar counter}}`，但incvar会返回新值。

---

### 递减 (Decrement)
使用 `--` 将数字变量减少 1：

* `{{.counter--}}` // 递减局部变量
* `{{$counter--}}` // 递减全局变量

类似 `{{decvar counter}}` 和 `{{decglobalvar counter}}`。但decvar会返回递减后的新值。

---

### 加法 / 追加 (Add)
使用 `+=` 为变量增加数值：

* `{{.score += 10}}` // 局部变量加 10
* `{{$total += 5}}`  // 全局变量加 5

等同于 `{{addvar::score::10}}` 和 `{{addglobalvar::total::5}}`。执行后返回空字符串。

如果两个操作数都不是数字，`+=` 运算符还支持将字符串追加到现有字符串变量末尾：
* `{{.myvar += {{noop}} | Second block}}` 
    * 如果原变量内容为 "Content"，则结果为 "Content | Second block"。
    * 使用 `{{noop}}` 可以保留空格，否则空格会被自动修剪（Trim）。

---

### 减法 (Subtract)
使用 `-=` 从变量中减去数值：

* `{{.health -= 10}}` // 局部变量减 10
* `{{$points -= 5}}`  // 全局变量减 5

等同于 `{{addvar::score::10}}`（使用负数/反转值）。返回空字符串。如果变量不是有效的数字，系统将记录警告且变量保持不变。

---

### 逻辑或 (Logical Or)
使用 `||` 在变量为**假值（falsy）**（空字符串、0、false）时提供回退值：

* `{{.name || Anonymous}}`  // 如果 .name 为空或假值，返回 "Anonymous"
* `{{$setting || default}}` // 如果 $setting 为假值，返回 "default"

如果变量为真值，则返回变量值；否则返回回退值。回退值采用**延迟计算（Lazy evaluation）**，仅在需要时才会求值。

---

### 空值合并 (Nullish Coalescing)
使用 `??` 仅在变量**不存在**时提供回退值：

* `{{.name ?? Guest}}`      // 仅当 .name 未定义时返回 "Guest"
* `{{$config ?? default}}`  // 仅当 $config 不存在时返回 "default"

与 `||` 不同，只要变量存在，即使它是假值（空字符串、0、false），也会返回该变量值。回退值采用延迟计算。

---

### 逻辑或赋值 (Logical Or Assign)
使用 `||=` 仅在变量当前为**假值**时对其进行设置：

* `{{.name ||= Anonymous}}` // 如果 .name 为假值，设置变量值 "Anonymous"
* `{{$count ||= 0}}`        // 如果 $count 为假值，设置 "0"

如果变量已经是真值，则不作修改。

---

### 空值合并赋值 (Nullish Coalescing Assign)
使用 `??=` 仅在变量**不存在**时对其进行设置：

* `{{.name ??= Guest}}`     // 仅当 .name 未定义时设置变量值 "Guest"
* `{{$config ??= default}}` // 仅当 $config 不存在时设置 "default"

与 `||=` 不同，如果变量已经存在，即使是假值也会被保留。

---

### 等于 (Equals)
使用 `==` 比较变量值与另一个值：

* `{{.status == active}}`   // 如果 .status 等于 "active" 返回 "true"，否则返回 "false"
* `{{$mode == dark}}`       // 如果 $mode 等于 "dark" 返回 "true"，否则返回 "false"

执行**字符串比较**，并返回字面量字符串 "true" 或 "false"。它会将不存在的变量、null 变量和空变量视为相同。
常用于 `{{if}}` 条件判断：
`{{if {{.status == active}} }}当前为激活模式{{/if}}`

---

### 不等于 (Not Equals)
使用 `!=` 比较变量值是否不相等：

* `{{.status != inactive}}` // 如果 .status 不等于 "inactive" 返回 "true"
* `{{$mode != light}}`      // 如果 $mode 不等于 "light" 返回 "true"

执行字符串比较。会将不存在、null 和空变量视为相同。

---

### 大于 (Greater Than)
使用 `>` 检查变量的数值是否大于另一个值：

* `{{.score > 50}}` // 如果 .score 大于 50 返回 "true"
* `{{$level > 5}}`  // 如果 $level 大于 5 返回 "true"

执行**数值比较**。常用于 `{{if}}` 条件判断。

---

### 大于或等于 (Greater Than or Equal)
使用 `>=` 检查变量的数值是否大于或等于另一个值：

* `{{.level >= 10}}`   // 如果 .level 至少为 10 返回 "true"
* `{{$points >= 100}}` // 如果 $points 为 100 或更多返回 "true"

---

### 小于 (Less Than)
使用 `<` 检查变量的数值是否小于另一个值：

* `{{.health < 20}}` // 如果 .health 低于 20 返回 "true"
* `{{$timer < 0}}`   // 如果 $timer 为负数返回 "true"

---

### 小于或等于 (Less Than or Equal)
使用 `<=` 检查变量的数值是否小于或等于另一个值：

* `{{.health <= 0}}`   // 如果 .health 为 0 或更低返回 "true"
* `{{$attempts <= 3}}` // 如果 $attempts 为 3 或更少返回 "true"
